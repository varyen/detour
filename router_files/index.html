Список доменов для проксирования

По одной записи на строку: домены, IP-адреса, CIDR-подсети. Комментарии через // (в начале строки или после записи). Строки без точки (заголовки секций) игнорируются.

Сайты-исключения: домены, IP-адреса и подсети из этого списка НЕ будут проксироваться. Весь остальной трафик пойдёт через VPN. Комментарии через // или #.

Маршрутизация по профилям

Режим запуска sing-box. Один инстанс — единый процесс разводит домены/подсети по разным профилям через route.rules (дёшево, +0 процессов). Несколько инстансов — отдельный процесс/порт/ipset на каждый целевой профиль (изоляция, но +RAM на каждый).

Управление группами

Группы — это просто строковая метка у профиля. Переименование меняет поле group у всех профилей в группе. Удаление группы НЕ удаляет профили — они становятся «без группы».

Подписки (per-group auto-refresh)

Каждая подписка привязана к группе профилей. По интервалу (по умолчанию 24ч) роутер сам забирает список, обновляет профили в группе и удаляет те, что пропали из подписки. Поддерживаемые форматы upstream-ответа: v2ray JSON, sing-box JSON, base64-обёртка с vless/trojan/vmess/ss ссылками, или просто список ссылок.
