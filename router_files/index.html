            Push-уведомления
          

          

            Браузер получит уведомление, когда активный VPN перестанет работать
            или окажется нерабочим сразу после подключения (функциональная
            проверка раз в 30–60 секунд), либо произойдёт авто-переключение.
            Работает, даже если вкладка панели закрыта.
          


          
          

            

              Push-уведомления доступны только когда панель открыта по
              HTTPS (например
              https://panel.example.com/detour/). По адресу http://IP браузер
              их не разрешает — откройте панель по защищённому домену.
            

          


          

            

              
              Включить push в этом браузере
            

            

            

              
            

          


          

            
            
            
          

        

      


      
      

        

          

            
            HTTPS-доступ к панели (Keenetic)
          

          

            Push-уведомления работают только когда панель открыта по HTTPS.
            На Keenetic не нужен отдельный сертификат Let's Encrypt — проще
            опубликовать панель через встроенный сервис KeenDNS: роутер сам
            выдаёт и продлевает валидный сертификат, и панель открывается по адресу
            вида https://detour.ваш-хост.keenetic.pro/detour/.
          


          

            
    
              
  1. 
                В веб-интерфейсе Keenetic включите KeenDNS
                (Параметры → Доменное имя). Появится имя вида
                ваш-хост.keenetic.pro.
              
    2. 
              
  2. 
                Откройте раздел
                «Доступ к веб-приложениям домашней сети» и нажмите
                «Добавить».
              
    3. 
              
  3. 
                Заполните запись:
                
      
                  
    • Доменное имя — поддомен, напр. detour
      • 
                  
    • 
                    IP-адрес — адрес этого роутера в домашней сети (напр.
                    192.168.1.1)
                  
      • 
                  
    • Протокол — HTTP
      • 
                  
    • 
                    Порт TCP — 8080 (порт панели
                    Detour)
                  
      • 
                
    
              
    4. 
              
  4. 
                Сохраните. Через ~минуту панель станет доступна по
                https://detour.ваш-хост.keenetic.pro/detour/.
              
    5. 
              
  5. 
                Откройте этот HTTPS-адрес, вернитесь сюда и включите push.
              
    6. 
            

          


          

            Запись делает панель доступной из интернета — задайте надёжный
            логин/пароль панели (⚙ → Сменить логин/пароль). Порт
            панели подставлен автоматически из настроек lighttpd этого роутера.
          


          

            
          

        

      


      
      

        

          

            
            HTTPS-сертификат (Let's Encrypt)
          

          

            Выпустить сертификат Let's Encrypt для домена, A-запись которого
            указывает на этот роутер, чтобы открывать панель по HTTPS (это нужно
            для push-уведомлений). Проверка — HTTP-01: на время выпуска
            порт 80 роутера должен быть доступен из интернета по этому домену.
          


          

            

          


          

            

          


          

            

              

                
                
              

              

                
                
              

            

            

              
            

            

          


          

            
          

        

      


      
      

        

          

            
            Файл подкачки (SWAP)
          

          

            На Keenetic панель и её сервисы (sing-box, lighttpd, dnsmasq) живут на
            USB-флешке, а оперативной памяти мало. Без файла подкачки стек под
            нагрузкой убивается по нехватке памяти — панель «ведёт себя
            нестабильно». Здесь можно создать SWAP-файл на флешке (он сам
            подключается при каждой загрузке) или удалить его.

            Аппаратное ускорение (offload)
          

          

            На этом роутере (QCA NSS/PPE) скорость LAN↔WAN держит аппаратный
            ускоритель. Иногда он «залипает» — форвардинг сваливается на CPU и
            скорость падает до ~100 Мбит, обычно до перезагрузки. Watchdog ловит это
            (всплеск softirq при простое прокси) и восстанавливает ускоритель
            без ребута.
          

          

            
Загрузка…

            

              
              
            

            
            

          

          

            Эффективность авто-kick подтверждается на реальном залипании — watchdog
            зафиксирует результат. Крайняя мера, если kick не помог, — перезагрузка
            роутера.

Список доменов для проксирования

              

                
                
              

            

            

              
              
            

            

              

                По одной записи на строку: домены, IP-адреса, CIDR-подсети.
                *.host.ru = все поддомены (как и просто
                host.ru). Комментарии через // или
                # (в начале строки или после записи). Строки без
                точки (заголовки секций) игнорируются.
              

              
            

            

              

                Сайты-исключения: домены, *.поддомены, IP-адреса и
                подсети из этого списка НЕ будут проксироваться.
                Весь остальной трафик пойдёт через VPN. Комментарии через
                // или #.
              

              
            

          

        


        
        

          

            

              
Маршрутизация по профилям

              

                
              

            


            

              Режим запуска sing-box. Один инстанс — единый
              процесс разводит домены/подсети по разным профилям через
              route.rules (дёшево, +0 процессов).
              Несколько инстансов — отдельный
              процесс/порт/ipset на каждый целевой профиль (изоляция, но +RAM на
              каждый).
            

            

              
              
            


            

              

                
                Keep-alive активного VPN:
                …
                
              

              

            


            

              Правила: выбери профиль, нажми «Добавить секцию», затем впиши
              домены / *.поддомены / IP / CIDR (по одному на строку)
              — как в обычном списке доменов. Каждая секция начинается заголовком
              // === route:ID ===. Хосты из правил автоматически
              попадают в проксируемый набор.

Загрузка...

            
            

              
              
            

            

              Логи дублируются в системный журнал через logger. Файлы
              логов при этом сохраняются — этот просмотрщик не меняется. Чтобы они

Домены Zapret/2 (общий список для zapret и zapret2)

              

                
                
              

            

            

              Эти домены идут НАПРЯМУЮ через WAN с DPI-трюками (минуя sing-box
              VLESS). Полезно для YouTube/Discord/LinkedIn без удалённого
              прокси, и для проверки tpws-пресета на конкретных сайтах. Форматы:
              домены, *.поддомены, IP, CIDR; комментарии
              // и #.
            

            
            

          

        


        

        
        

          

            

              
Приоритетный hosts-файл (DNS-обход)

              

                
                
              

            

            

              Подключает внешний hosts-файл (например GeoHide DNS)
              к внутреннему резолверу роутера (dnsmasq, addn-hosts)
              — записи получают приоритет над upstream-DNS.

              Это параллельный обход: трафик к таким доменам идёт
              НАПРЯМУЮ на IP из файла, минуя ваш sing-box / zapret. Домены из
              ваших списков sing-box (proxy-domains.list, а в
              режиме all-except —
              whitelist-domains.list) по умолчанию исключаются,
              чтобы не переопределять вашу маршрутизацию.
            


            

              ⚠ На этой платформе нет dnsmasq — функция недоступна.

📡 Статусы VPN

            

              
            

          

          

            Все профили VPN, отсортированные по пингу (меньше — выше). Пинги
            обновляются в фоне; можно обновить пинг конкретного сервера или все
            сразу, а также сделать любой VPN активным прямо из таблицы.
          

          

            
            
            Сортировка — по возрастанию пинга. Зелёная строка — активный
              VPN.
          

          
          

            
            
            
            
            
            
          

          

          

            

              По одной цели на строку: Название|https://адрес (название
              необязательно). Профиль считается «рабочим», только если открылись
              ВСЕ цели. Пусто = значения по умолчанию (YouTube + видео-CDN + Google).

              Подписки
            

            

              
            

          

          

            Группа — это строковая метка у профиля: переименование меняет поле
            group у всех профилей группы, удаление группы НЕ
            удаляет профили (они становятся «без группы»). К любой группе можно
            привязать подписку — по интервалу (по умолчанию 24ч) роутер
            сам забирает список, обновляет профили группы и удаляет пропавшие.
            Форматы upstream-ответа: v2ray JSON, sing-box JSON, base64-обёртка с
            vless/trojan/vmess/ss ссылками или просто список ссылок.
          

          

            
            
            
            
          

          

          


          
          

            

              Подписка группы
            

            

              

                
                
                

                  Только a-z, 0-9, ._-. Автоподставится из группы.
